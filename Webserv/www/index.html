Webserv

A lightweight HTTP server built in C++ as a group project at 42 Berlin.

About

The web is the system that allows us to access information, applications, and services online. It is built on two sides working together:

The Server

A program that listens for requests and responds with the needed data. It runs the backend: the code that determines how data is stored, processed, and updated.

The Client

The interface the user sees and interacts with, such as a website or app. It runs the frontend: the code that defines the layout and handles user interactions.

Webserv is an HTTP server written in C++ for the 42 Berlin curriculum. Even though our assignment focused only on building a server, we built with both server and client in mind:

About our Webserv

Built under strict constraints (C++98, no external libraries, non-blocking I/O), our server shows how modern servers like NGINX work:

  • Non-blocking I/O with socket programming and an event-driven loop.
  • Support for HTTP methods: GET, POST, DELETE.
  • Serves static files, manages file uploads, and executes CGI scripts.
  • Custom error pages and per-path configuration blocks.
  • Multiple ports to serve different sites at the same time.
About our Frontend

We added a frontend to make the demo complete during evaluation. This layer lets us:

  • Verify the server works end to end.
  • Expose hidden bugs or hardcoded behavior.
  • Show how methods and responses behave in practice.
  • Give evaluators a clear way to test, regardless of experience.
